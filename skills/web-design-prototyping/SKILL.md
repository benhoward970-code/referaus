# Web Design Prototyping Skill

## WHEN TO USE
- Creating visual mockups and prototypes for web interfaces
- Designing layouts before implementation
- Client presentations and design approval
- Rapid prototyping for UI concepts
- When Figma/design tools aren't accessible

## WHEN NOT TO USE
- Final production code (this is for prototyping only)
- Complex interactive applications
- When actual design tools (Figma, Sketch) are available

## DEPENDENCIES
- HTML/CSS knowledge
- Modern browser for viewing
- Optional: Design frameworks (Tailwind, Bootstrap)

## APPROACH

### Method 1: HTML/CSS Prototypes
- Create visual mockups using HTML and CSS
- Use modern CSS features (Grid, Flexbox, animations)
- Include realistic content and data
- Mobile-responsive layouts

### Method 2: Design Documentation
- Create detailed design specifications
- Color palettes, typography, spacing systems
- Component documentation with code examples
- Layout wireframes with annotations

### Method 3: Interactive Prototypes
- Add JavaScript for basic interactions
- Hover states, animations, form interactions
- Navigation between prototype pages
- Realistic data and content

## WORKFLOW

### 1. Requirements Gathering
- Understand design goals and constraints
- Identify target users and devices
- Define color schemes and branding
- List required pages and components

### 2. Design System Creation
- Color palette definition
- Typography scale and fonts
- Spacing and grid systems
- Component library documentation

### 3. Layout Prototyping
- Create wireframe layouts
- Add realistic content and imagery
- Implement responsive breakpoints
- Test across different screen sizes

### 4. Interaction Design
- Add hover and focus states
- Implement animations and transitions
- Create navigation flows
- Add form interactions

### 5. Documentation
- Create design specifications
- Document component usage
- Provide implementation guidelines
- Include accessibility notes

## OUTPUT ARTIFACTS
- **Prototype HTML files** - Visual mockups
- **Design system CSS** - Reusable styles
- **Design specifications** - Implementation guide
- **Component documentation** - Usage guidelines

## SUCCESS CRITERIA
- Visually accurate representation of final design
- Responsive across all target devices  
- Professional appearance and interactions
- Clear documentation for implementation
- Client/stakeholder approval of design direction

## TOOLS AND TECHNIQUES

### CSS Frameworks
- **Tailwind CSS** - Utility-first framework
- **Bootstrap** - Component-based framework
- **Custom CSS** - Full control over styling

### Design Elements
- **Typography** - Web fonts, hierarchy, readability
- **Color Systems** - Accessible palettes, semantic colors
- **Layout Grids** - Responsive, flexible layouts
- **Components** - Reusable UI elements

### Prototyping Features
- **Animations** - CSS transitions and keyframes
- **Interactions** - JavaScript for dynamic behavior
- **Responsive Design** - Mobile-first approach
- **Accessibility** - WCAG compliance considerations

## EXAMPLE WORKFLOW

```
1. Create design brief and requirements
2. Build color palette and typography system
3. Create layout wireframes with HTML structure
4. Add styling with CSS (or Tailwind)
5. Implement responsive breakpoints
6. Add interactions with JavaScript
7. Test across devices and browsers
8. Document design decisions and specifications
9. Present prototype for approval
10. Provide implementation guidelines
```

## LIMITATIONS
- Not a replacement for professional design tools
- Limited to web-based interactions
- Requires coding knowledge
- Less collaborative than visual design tools

## WHEN TO UPGRADE
- Move to Figma/Sketch for complex design systems
- Use professional tools for team collaboration
- Consider design handoff tools for developer workflow
